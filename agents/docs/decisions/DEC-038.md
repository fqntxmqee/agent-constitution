# DEC-038: 移除进展跟进智能体，职责由银河导航员接管

**决策编号**: DEC-038  
**决策标题**: 移除进展跟进智能体 (progress-tracking)，职责由银河导航员接管  
**决策日期**: 2026-03-26  
**决策类型**: Type-A (Major 变更)  
**决策状态**: ✅ 已生效（用户授权跳过冷静期）

| 冷静期 | 3 天（可用户授权跳过） |

---

## 🎯 决策背景

### 现状 (V3.15.0)
- 银河导航员作为 Hub 负责任务分发
- progress-tracking 作为独立智能体负责进度跟踪
- 存在职责重叠：Hub 天然应监控 Spoke 状态

### 问题
1. **职责重叠**: 银河导航员已掌握任务状态，progress-tracking 需二次收集
2. **效率损失**: 多一层智能体调用，增加延迟和 token 消耗
3. **架构冗余**: Hub-Spoke 模式下，进度跟踪是 Hub 的内置职责

### 触发事件
用户提出移除进展跟进智能体，选择由银河导航员直接接管（选项 A）

---

## 📐 决策方案

### 核心变更
1. **移除** `agents/constitution/progress-tracking/` 智能体目录
2. **合并** 进度跟踪职责至银河导航员
3. **更新** 宪法规范文件（CONSTITUTION.md, TEAM_ROLES.md, GALAXY_NAVIGATOR.md）
4. **归档** 历史进度报告

### 新架构
```
银河导航员 (Hub) [任务分发 + 进度跟踪]
├── 需求澄清
├── 需求理解
├── 需求解决
├── 需求验收
├── 需求交付
├── 审计 [监督进度异常]
├── 总结反思
└── 调试专家
```

### 进度跟踪机制
- **任务状态更新**: 银河导航员在分发任务时直接更新 `.tasks/index.md`
- **周期汇报**: 内置心跳检查（每 3-5 分钟）
- **停滞检测**: 超时任务自动标记 + 审计智能体监督
- **通知渠道**: 银河导航员直接发送飞书/聊天通知
