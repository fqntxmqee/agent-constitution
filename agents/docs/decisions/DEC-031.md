## 决策摘要
引入 Story File 结构，解决多智能体协作中的上下文丢失问题。

## 背景
当前宪法规范中，OpenSpec 规约完整但上下文连续性不足——每个智能体启动时重新加载规约，缺乏"开发过程记忆"，导致决策漂移、信息丢失、重复劳动。

## 选项对比
| 选项 | 方案 | 优点 | 缺点 |
|------|------|------|------|
| A | 维持现状 | 无额外工作 | 上下文丢失问题持续 |
| B | Story File 结构 | 上下文连续、可追溯 | 需更新所有 AGENTS.md |
| C | 仅增强 prompt | 改动小 | 上下文仍分散在聊天记录 |

## 决策结果
**采纳选项**: B

## 决策内容
- 在 `project/{项目名}/changes/{需求名}/` 下新增 `story/` 目录
- `story/state.md` 作为上下文传递的核心载体
- 各智能体启动时读取 state.md，完成后更新
