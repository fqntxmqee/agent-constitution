## 决策摘要
引入 C/B/A/S 四级需求复杂度评级，按复杂度选择差异化开发路径。

## 背景
当前宪法按 Type-A/B/C 分类规范变更，不适合需求开发场景。简单需求过度工程（完整走5阶段），复杂需求保障不足。

## 解决方案
| 复杂度 | 路径 | 冷静期 |
|--------|------|--------|
| C级（简单） | 快速流（跳过理解智能体） | 无 |
| B级（普通） | 标准流 | 无 |
| A级（复杂） | 完整流 + 24h冷静期 | 24h |
| S级（企业） | 完整流 + 外部评审 + 审计增强 | 3天 |
