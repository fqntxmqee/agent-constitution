## 决策摘要
所有需要用户确认的文档（澄清提案、规约蓝图文书、验收报告等），必须本地有 + 飞书有 + **飞书链接发送给用户**，才能进入下一阶段。

## 背景
用户反馈：当前宪法流程中，规约同步到飞书后用户无法便捷查阅，导致流程不透明。

## 解决方案
更新 `SPEC_OpenSpec_Sync.md`（V3.12.0），新增"双重保活原则"和"链接发送规范"：

1. **双重保活**：本地有 + 飞书有 + 链接发送给用户，三者同时满足
2. **主会话行为约束**：未发送飞书链接给用户，禁止进入下一阶段
3. **用户确认节点规范**：每个节点必须列出所有文档及飞书链接，等待用户明确回复

## 主会话禁止行为
| 禁止 | 说明 |
|------|------|
| ❌ 跳过链接发送 | 未发送飞书链接给用户，禁止进入下一阶段 |
| ❌ 本地替代 | 本地文件存在但飞书未同步，禁止进入下一阶段 |
| ❌ 仅口头确认 | 必须有实际的飞书链接，用户确认「可以」 |
