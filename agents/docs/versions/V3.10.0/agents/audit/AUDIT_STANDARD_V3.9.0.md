## 📖 第一章 总则

### 1.1 审计目标

本审计规范旨在建立系统化、标准化的智能体协同系统审计机制，确保：

1. **安全性** - 防止未授权访问、数据泄露、恶意操作
2. **合规性** - 遵循宪法规范、工作流程、安全策略
3. **可靠性** - 系统稳定运行、故障可追溯、恢复可验证
4. **透明度** - 操作可审计、决策可解释、责任可追溯

### 1.2 审计原则

| 原则 | 说明 |
|------|------|
| **独立性** | 审计智能体独立于执行智能体，直接向用户报告 |
| **客观性** | 基于规则和证据，不依赖主观判断 |
| **完整性** | 覆盖所有智能体、所有操作、所有通道 |
| **及时性** | 实时监控 + 定期审计 + 事件触发 |
| **可追溯** | 所有审计记录可查询、可验证、不可篡改 |

## 📖 第二章 审计组织架构

### 2.1 审计智能体定位

```
                    ┌──────────────┐
                    │    用户      │
                    │   (决策者)   │
                    └──────┬───────┘
                           │ 报告/告警
                    ┌──────▼───────┐
                    │  规则守护者  │
                    │   (审计)     │
                    │  🛡️ 独立    │
                    └──────┬───────┘
                           │ 监控/熔断
        ┌──────────────────┼──────────────────┐
        │                  │                  │
┌───────▼────────┐ ┌──────▼───────┐ ┌────────▼────────┐
│  银河导航员    │ │  各需求智能体 │ │  支撑智能体     │
│  (协调者)      │ │  (执行者)    │ │  (进度/总结)   │
└────────────────┘ └───────────────┘ └─────────────────┘
```

### 2.2 审计权限

| 权限 | 说明 | 限制 |
|------|------|------|
| **读取权** | 可读取所有智能体的指令流、产出、日志 | 无限制 |
| **监控权** | 可实时监控所有智能体的运行状态 | 无限制 |
| **熔断权** | 可暂停任何智能体的执行 | 需用户确认恢复 |
| **报告权** | 可直接向用户发送审计报告 | 无限制 |
| **执行权** | ❌ 不可执行 | 禁止修改其他智能体产出 |

### 2.3 审计独立性保障

- 审计智能体配置独立于其他智能体
- 审计日志存储独立于业务日志
- 审计决策不受其他智能体影响
- 审计报告直接送达用户

### 4.3 熔断机制

#### 4.3.1 熔断条件

| 条件 | 说明 | 响应 |
|------|------|------|
| 无规约开发 | 缺少 proposal/design/tasks | 立即熔断 |
| runtime 违规 | 使用 subagent 执行开发 | 立即熔断 |
| 跳过验收 | 无验收报告直接交付 | 立即熔断 |
| 高危操作 | 下载/执行未知代码 | 立即熔断 |
| 数据泄露 | 敏感数据外传 | 立即熔断 |
| 越权操作 | 访问未授权资源 | 立即熔断 |
