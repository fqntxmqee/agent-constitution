# 决策记录：移除 ACP 强制要求 (V3.12.0 → V3.13.0)

**决策编号**: DEC-041

## 📋 决策摘要

**决策内容**: 移除宪法规范中 `runtime="acp"` 的强制要求，允许智能体根据任务复杂度自主选择执行方式。

**决策原因**: 
- 强制 ACP 在某些简单场景下增加不必要开销
- 智能体需要根据任务复杂度灵活选择执行方式
- Subagent 在简单任务中更高效

**保留原则**: 
- 继续禁止主会话直接 `write` 业务代码（铁律）
- 复杂任务仍推荐使用 ACP

---

## 🎯 决策背景

### 问题陈述

V3.7.2 引入的 `runtime="acp"` 强制规范在实际运行中暴露以下问题：

1. **过度工程化**: 简单配置修改、单文件更新等任务被迫使用 ACP，启动开销大
2. **灵活性缺失**: 智能体无法根据任务特点选择最优执行方式
3. **效率损失**: Subagent 在简单任务中的快速启动优势被强制禁止

### 变更目标

1. 移除强制约束，赋予智能体自主选择权
2. 保留核心质量保障（禁止主会话直接 write 业务代码）
3. 建立执行方式选择指南，帮助智能体做出合理决策

---

## 📊 决策分析

### 执行方式对比

| 执行方式 | 适用场景 | 启动时间 | 上下文能力 | 工具能力 |
|---------|---------|---------|-----------|---------|
| `runtime="acp"` | 复杂开发、多文件修改、代码审查 | ~5-10s | 完整项目上下文 | Cursor CLI 全能力 |
| `runtime="subagent"` | 简单任务、单文件修改、配置调整 | ~1-2s | 有限上下文 | OpenClaw 工具 |
| 直接工具调用 | 单一操作 | 即时 | 无 | 单个工具 |

### 决策矩阵

| 任务类型 | 推荐执行方式 | 理由 |
|---------|-------------|------|
| 多文件代码修改 | ACP | 需要完整上下文、代码审查 |
| 单文件配置修改 | Subagent | 简单、快速、无需审查 |
| 文档更新 | Subagent | 无需代码能力 |
| 测试运行 | ACP | 需要完整环境 |
| Git 操作 | Subagent 或 ACP | 复杂度决定 |
| 需求理解（生成规约） | ACP | 需要读取项目结构 |
| 需求验收（测试 + 审查） | ACP | 需要完整验证能力 |
