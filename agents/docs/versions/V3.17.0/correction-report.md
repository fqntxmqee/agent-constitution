### 2.2 双模式 SLA（本次新增）

在"1 分钟 SLA 效果对比"与"协同方式"之间新增 **"⏱ 双模式 SLA：快速响应（1 分钟）vs 标准响应（5 分钟）"** 章节，包含：

#### 新增内容

1. **双模式总览表**：快速响应 vs 标准响应的对比（适用场景/目标响应/确认时限/超时熔断/派发模式/回报频率）

2. **快速响应模式（1 分钟）**：
   - 适用场景：C 级任务、独立任务、查询类任务、修复类任务
   - 时间分解：派发 ≤10s + 确认 ≤10s + Spawn ≤10s + 执行 ≤20s + 回报 ≤10s = ≤60s

3. **标准响应模式（5 分钟）**：
   - 适用场景：A/S 级任务、流水线任务、大总管配合任务、深度分析任务、多智能体协同
   - 时间分解：派发 ≤30s + 确认 ≤30s + 计划 ≤60s + 执行 ≤3min + 回报 ≤60s = ≤5min

4. **智能体选择策略**：按任务类型推荐模式（查询/修复 → 快速；澄清/理解/解决/验收/交付/审计 → 标准）

5. **超时熔断机制（两级超时 + 三级降级）**：
   - 两级超时：一级（确认超时）/ 二级（执行超时）
   - 三级降级：30% 无进展 / 60% 无进展 / 主动取消

### 3.2 每个智能体新增内容

**9 大核心智能体**的 `AGENTS.md` 在"响应 SLA"和"响应 SLA 配置"章节中均已更新，新增：

#### 快速响应模式（1 分钟）
- ⏱ **10 秒内确认**：收到任务后 10 秒内回复"已收到"
- ⏱ **60 秒内完成**：对应复杂度级别任务 60 秒内完成
- ⏱ **超时降级**：60 秒未完成则降级为简化流程并回报进度

#### 标准响应模式（5 分钟）
- ⏱ **30 秒内确认**：收到任务后 30 秒内回复"已收到 + 初步计划"
- ⏱ **5 分钟内完成**：A/S 级任务 5 分钟内产出完整结果
- ⏱ **分段回报**：每分钟回报一次进度（1min 计划 / 3min 执行 / 4.5min 终稿）
- ⏱ **超时熔断**：2min 无进展 → 降级；5min → 熔断上报

#### 响应 SLA 配置表（更新后）

| 模式 | 确认时限 | 完成时限 | 超时处理 |
|------|---------|---------|---------|
| **快速响应（1 分钟）** | ≤10s | ≤60s | 60s 超时 → 熔断 |
| **标准响应（5 分钟）** | ≤30s | ≤5 分钟 | 2min 降级 / 5min 熔断 |

| 指标 | 要求 |
|------|------|
| **确认响应** | ≤10s（快速）/ ≤30s（标准）内发送确认 ACK |
| **快速任务完成** | ≤60s 内完成并回报（对应级别） |
| **标准任务完成** | ≤5 分钟内完成并完整回报（A/S 级） |
| **超时降级** | 30s 无进展 → 返回处理中状态；60s/5min 超时 → 熔断 |
| **降级内容** | `{ status: "processing/degraded", partial: true, suggestion: "..." }` |
