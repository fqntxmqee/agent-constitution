# 流程优化建议：宪法规范升级合规性加固

> **编号**：OPT-001
> **日期**：2026-03-22
> **状态**：✅ 已实施（2026-03-22 14:52）
> **基于**：VIP-001 流程违规报告
> **适用范围**：V3.12.0 升级落地

---

## 一、问题根因总结

| 根因 | 说明 |
|------|------|
| **R1：缺乏强制触发** | 修改宪法文件时，没有机制强制检查「是否在规范流程内」 |
| **R2：spawn 成本过高** | summary-reflection spawn 需要完整上下文理解，导致跳过 |
| **R3：无轻量入口** | 没有一个「说一句话就能启动规范升级流程」的命令 |
| **R4：行为惯性** | 「先做事再问」在规范执行场景被误用 |

---

## 二、优化方案

### OPT-A：AGENTS.md 新增强制检查点

**位置**：`AGENTS.md` > `## 宪法规范执行`

**新增内容**：

```markdown
## 宪法规范执行（V3.12.0 新增）

### 强制检查点

涉及以下类型的文件创建/修改时，**必须**在执行前完成：

1. **识别触发**：检测到修改 `CONSTITUTION.md`、`AGENTS.md`、`ETHOS.md`、任意 `*/AGENTS.md`
2. **立即查阅**：`agents/docs/specs/constitution/upgrade/ITERATION_PROCESS.md`
3. **选择路径**：
   - 正式升级路径：spawn `summary-reflection` 生成提案（适用于功能变更、架构调整）
   - 紧急修复路径：spawn `audit` + 用户确认 + 当场补充文档（适用于错别字、格式修正）
4. **禁止跳过**：无论需求多小，禁止主会话直接修改宪法文件

### 轻量升级命令

用户可以说：
- "帮我升级宪法规范 XXX"
- "我想调整 ETHOS.md"
- "有个流程优化建议"

主会话收到后，执行：
```
spawn summary-reflection（轻量 prompt："用户想升级宪法规范：[用户描述]，请生成规范升级提案"）
```

### 判断标准

| 场景 | 路径 |
|------|------|
| 新增智能体 | 正式升级（ITERATION_PROCESS.md） |
| 删除/合并智能体 | 正式升级 + 冷静期 |
| 流程变更（不涉及文件修改） | 正式升级 |
| 错别字/格式修正 | 紧急修复路径（audit 现场确认） |
| ETHOS/SOUL 人格调整 | 正式升级 |
| 迁移文件位置（无内容变更） | 紧急修复路径 |

---

### OPT-B：audit 智能体新增检查项

**位置**：`agents/constitution/audit/AGENTS.md` > 检查清单

**新增检查**：

```markdown
### C. 宪法文件变更检查（V3.12.0 新增）

#### 时间窗口：最近 24 小时内

检测以下文件的变更：
- `CONSTITUTION.md`
- `AGENTS.md`（workspace 根目录）
- `ETHOS.md`
- `agents/constitution/*/AGENTS.md`

#### 检查逻辑

1. **目录检查**：变更文件是否在 `agents/docs/versions/V{version}/constitution/upgrade-to-V{version}/` 目录下？
2. **提案检查**：是否存在对应的 `DECISION_LOG.md`？
3. **用户确认检查**：DECISION_LOG.md 中是否有用户确认记录？

#### 判定结果

| 结果 | 条件 | 处理 |
|------|------|------|
| ✅ 合规 | 文件在正确目录 + 有 DECISION_LOG + 有用户确认 | 无需处理 |
| ⚠️ 待补充 | 文件在正确目录 + 无 DECISION_LOG | 提醒主会话补充 |
| 🔴 违规 | 宪法文件被修改但不在正确目录 | 标记违规 + 触发告警 |

#### 违规处理

```
发现违规：
1. 立即通知主会话："检测到宪法文件被修改但未走规范流程，详见 agents/docs/versions/V{version}/constitution/upgrade-to-V{version}/PROCESS_VIOLATION_REPORT.md"
2. 要求主会话在 24h 内补充 DECISION_LOG.md
3. 如果主会话已在补充中，标记为"已发现，待纠正"
4. audit 报告记录本次违规
```
