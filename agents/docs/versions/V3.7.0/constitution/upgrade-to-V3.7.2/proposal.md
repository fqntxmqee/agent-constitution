#### 第四章 4.5 新增章节
```markdown
## 4.5 宪法规范迭代流程

### 迭代触发条件
1. 每日总结：每日 23:00 触发
2. 里程碑达成：重大任务完成后触发
3. 审计发现：审计智能体发现规范缺陷时触发
4. 用户请求：用户明确要求更新规范时触发

### 迭代分析维度
| 维度 | 分析内容 | 数据来源 |
|------|---------|---------|
| 合规性 | 是否有违规操作 | 审计报告 |
| 效率 | 是否有流程瓶颈 | 进展报告 |
| 质量 | 是否有验收失败 | 验收报告 |
| 技能 | 是否有技能缺失 | 技能目录 + 任务记录 |
| 流程 | 是否有流程缺陷 | 会话日志 + 验收报告 |

### 迭代提案格式
- 变更概览（章节、变更类型、变更摘要）
- 变更对比（变更前 vs 变更后）
- 影响分析（影响智能体、技能、流程）
- 用户确认项（必须用户确认的变更项）

### 用户确认流程
1. 提案提交：总结反思智能体提交《宪法规范迭代提案》
2. 用户审查：用户审查变更内容、影响分析
3. 用户确认：用户确认变更项（勾选确认框）
4. 审计监督：审计智能体验证变更符合规范
5. 正式生效：更新宪法规范版本号，正式生效

### 版本管理
- 版本号规则：V{主版本}.{次版本}.{修订号}
- 主版本：重大架构变更（如新增智能体）
- 次版本：功能增强（如新增技能、流程优化）
- 修订号：文字修订、错别字修正
- 变更日志：agents/docs/specs/constitution/CHANGELOG.md
```
