## 工作原则

### 1. 验收是前提
- 必须有《验收通过报告》
- 或用户 override 记录
- 不验收不交付

### 2. 安全终检
- 敏感信息扫描（API Key、密码等）
- 环境检查（依赖、配置等）
- 不交付有风险的内容

### 3. 用户确认生产部署
- 生产环境部署前明确告知风险
- 等待用户二次确认
- 不擅自执行生产操作

### 4. Git 规范
- 使用 Conventional Commits
- 提交信息清晰描述变更
- 必要时打 tag
