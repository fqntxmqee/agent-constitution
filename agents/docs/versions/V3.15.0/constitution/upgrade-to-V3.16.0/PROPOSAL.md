# 宪法规范升级提案：V3.15.0 → V3.16.0

**提案类型**: Type-A (Major 变更 - 删除智能体)  
**提案日期**: 2026-03-26  
**提案智能体**: 银河导航员 🧭  
**触发事件**: 移除进展跟进智能体 (progress-tracking)，职责由银河导航员接管

---

## 📋 变更摘要

| 项目 | 当前状态 (V3.15.0) | 变更后状态 (V3.16.0) |
|------|-------------------|---------------------|
| 智能体数量 | 9+1 (9 个子智能体 + 银河导航员) | 8+1 (8 个子智能体 + 银河导航员) |
| 进展跟进智能体 | ✅ 独立智能体 | ❌ 移除 |
| 进度跟踪职责 | progress-tracking 负责 | 银河导航员直接接管 |
| 任务文件结构 | `.tasks/index.md` + 进度报告 | `.tasks/index.md` + 银河导航员内置跟踪 |

---

## 🎯 变更原因

### 1. 架构简化
- V3.15.0 引入 Hub-Spoke 模式后，银河导航员已作为 Hub 统一分发任务
- 进度跟踪本质是 Hub 的天然职责（监控 Spoke 状态）
- 独立 progress-tracking 智能体造成职责重叠

### 2. 效率优化
- 减少一层智能体调用，降低延迟
- 银河导航员可直接在任务分发时更新状态，无需等待进度报告
- 减少 token 消耗（少一个智能体上下文）

### 3. 职责清晰
- 银河导航员：任务分发 + 进度跟踪 + 状态汇报（完整 Hub 职责）
- 审计智能体：独立监督（含进度异常审计）
- 避免"催更小助手"与银河导航员的边界模糊

## 🔄 迁移方案

### 1. 职责迁移

**原 progress-tracking 职责** → **银河导航员**

| 职责 | 执行方式变更 |
|------|-------------|
| 监控任务状态 | 银河导航员在任务分发时直接更新 `.tasks/index.md` |
| 周期进度汇报 | 银河导航员内置心跳检查（每 3-5 分钟） |
| 停滞检测与报警 | 银河导航员检测 + 审计智能体监督 |
| 飞书通知 | 银河导航员直接发送 |

### 2. 数据迁移

```
原路径：agents/constitution/progress-tracking/reports/
新路径：agents/constitution/progress-tracking/ARCHIVED/ (备份)
```

**保留文件**:
- `.tasks/index.md` (格式不变)
- 历史进度报告 (归档)

### 3. 流程更新

**原流程**:
```
子智能体 → progress-tracking → 银河导航员 → 用户
```

**新流程**:
```
子智能体 → 银河导航员 → 用户
              ↓
         审计智能体 (监督)
```
