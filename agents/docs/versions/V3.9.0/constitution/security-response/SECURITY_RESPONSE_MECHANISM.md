# 安全问题响应机制

**版本**: V3.10.0  
**类型**: 宪法子规范  
**生效日期**: 2026-03-15

---

## 1. 问题等级定义

### 1.1 四级分类

| 等级 | 定义 | 示例 |
|------|------|------|
| **Critical** | 严重安全问题，立即影响系统安全 | 未授权访问、数据泄露风险、提示注入 |
| **High** | 高优先级问题，影响系统正常运行 | 服务中断、性能严重下降 |
| **Medium** | 中等优先级问题，影响部分功能 | 功能缺陷、用户体验下降 |
| **Low** | 低优先级问题，轻微影响 | 界面问题、文档错误 |

### 1.2 等级判定规则

1. **安全相关问题** → 优先判定为 Critical/High
2. **服务可用性** → 优先判定为 High/Critical
3. **功能性问题** → 判定为 Medium
4. **体验性问题** → 判定为 Low

---

## 2. 响应时限

### 2.1 时限表

| 问题等级 | 响应时限 | 修复时限 | 升级触发 |
|----------|----------|----------|----------|
| Critical | 1 小时 | 24 小时 | 超过 24 小时 |
| High | 4 小时 | 72 小时 | 超过 72 小时 |
| Medium | 24 小时 | 7 天 | 超过 7 天 |
| Low | 7 天 | 30 天 | 超过 30 天 |

### 2.2 时序图

```
首次发现 ──→[响应时限]──→[修复时限]──→[升级]
   ↓            ↓            ↓           ↓
  t=0      t=1h/4h/24h   t=24h/72h    触发告警
              /7d        /7d/30d
```

---

## 3. 升级机制

### 3.1 自动升级条件

| 问题等级 | 升级条件 | 升级操作 |
|----------|----------|----------|
| Critical | 超过 24 小时未修复 | 标记为已升级 + 告警 |
| High | 超过 72 小时未修复 | 标记为已升级 + 告警 |
| Medium | 超过 7 天未修复 | 标记为已升级 + 告警 |
| Low | 超过 30 天未修复 | 标记为已升级 + 告警 |

### 3.2 升级记录

升级时记录：
- 升级触发时间
- 持续审计次数
- 问题当前状态

---

## 4. 告警机制

### 4.1 告警触发条件

| 问题等级 | 告警类型 | 触发时机 |
|----------|----------|----------|
| Critical | 立即告警 | 问题发现时 |
| High | 预警 | 超过 24 小时未修复 |
| Medium | 提醒 | 超过 7 天未修复 |
| Low | 常规 | 超过 30 天未修复 |

### 4.2 告警内容

告警消息必须包含：
- 问题 ID 和描述
- 问题等级
- 首次发现时间
- 当前持续时间
- 审计次数

---

## 5. 跟踪机制

### 5.1 跟踪数据

| 字段 | 说明 |
|------|------|
| issue_id | 问题唯一标识 |
| first_detected_at | 首次发现时间戳 |
| detection_count | 审计检测次数 |
| severity | 问题等级 |
| status | 状态（open/in_progress/escalated/resolved） |
| escalated_at | 升级时间戳（可选） |
| resolved_at | 修复时间戳（可选） |

### 5.2 状态流转

```
open → in_progress → resolved
   ↓        ↓
escalated
```

## 6. 审计智能体职责扩展

### 6.1 问题检测

- [ ] 检测安全、合规、运行问题
- [ ] 对问题进行等级分类（Critical/High/Medium/Low）
- [ ] 记录问题详情和影响范围

### 6.2 问题跟踪

- [ ] 记录问题首次发现时间
- [ ] 跟踪问题修复状态
- [ ] 计算问题持续时间（以审计次数为单位）

### 6.3 升级判断

- [ ] 判断问题是否超过响应时限
- [ ] 触发自动升级流程
- [ ] 发送告警通知

### 6.4 报告生成

- [ ] 生成每日问题跟踪报告
- [ ] 生成问题趋势分析
- [ ] 记录问题修复历史

## 7. 可配置参数

| 参数 | 默认值 | 类型 | 说明 |
|------|--------|------|------|
| N_critical_response_hours | 1 | number | Critical 问题响应时限（小时） |
| N_critical_fix_hours | 24 | number | Critical 问题修复时限（小时） |
| N_high_response_hours | 4 | number | High 问题响应时限（小时） |
| N_high_fix_hours | 72 | number | High 问题修复时限（小时） |
| N_medium_response_hours | 24 | number | Medium 问题响应时限（小时） |
| N_medium_fix_days | 7 | number | Medium 问题修复时限（天） |
| N_low_response_days | 7 | number | Low 问题响应时限（天） |
| N_low_fix_days | 30 | number | Low 问题修复时限（天） |

## 8. 问题处理流程

```
┌──────────────┐
│  问题发现    │
└──────┬───────┘
       ↓
┌──────────────┐
│  等级分类    │ ← Critical/High/Medium/Low
└──────┬───────┘
       ↓
┌──────────────┐
│  记录时间    │ ← first_detected_at
└──────┬───────┘
       ↓
┌──────────────┐
│  响应判断    │ ← 是否超过响应时限？
└──────┬───────┘
       ↓
  ┌────┴────┐
  ↓         ↓
修复     升级判断
  ↓         ↓
更新状态   告警通知
  ↓         ↓
报告生成   用户确认
```
