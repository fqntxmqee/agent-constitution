# 审计检查清单规范

**版本号**: V1.0  
**生效日期**: 2026-03-12  
**关联决策**: DEC-016 至 DEC-020

---

## 一、Type-A 审计清单（100% 覆盖）

### A. 合规性检查
- [ ] 变更是否符合宪法核心原则
- [ ] 变更类型判定是否正确
- [ ] RFC 文档是否完整（10 节）
- [ ] 用户确认是否完整
- [ ] 冷静期是否满足（3 天）

### B. 技术检查
- [ ] 版本号是否符合 SemVer
- [ ] 影响分析是否完整
- [ ] 向后兼容性评估
- [ ] 回滚方案是否可行

### C. 文档检查
- [ ] RFC 文档完整性
- [ ] 变更日志更新
- [ ] 版本备份完成

---

## 二、Type-B 审计清单（核心项）

- [ ] 变更类型判定正确
- [ ] 提案文档完整（6 节）
- [ ] 用户确认完成
- [ ] 冷静期满足（24 小时）
- [ ] 版本号正确（MINOR 递增）

---

## 三、Type-C 审计（抽查）

- [ ] 变更类型判定正确
- [ ] Git Commit 规范

---

## 四、一票否决权

**触发条件**:
- [ ] 违反宪法核心原则
- [ ] 变更类型判定错误
- [ ] 冷静期未满足
- [ ] 用户确认缺失
- [ ] 回滚方案缺失

---

## 五、用户 Override 机制

**要求**:
- [ ] 用户书面确认
- [ ] 审计记录原因
- [ ] 文档标注「用户 Override」

---

## 六、审计 SLA

| 变更类型 | 完成时限 | 超时处理 |
|----------|---------|---------|
| **Type-A** | 24 小时 | 自动提醒 |
| **Type-B** | 4 小时 | 自动提醒 |
