# 宪法变更分类规范

**版本号**: V1.0  
**生效日期**: 2026-03-12  
**状态**: ✅ 已确认  
**关联决策**: DEC-007, DEC-008, DEC-009, DEC-010

---

## 一、变更分类定义

### 1.1 三级分类体系

| 类型 | 名称 | 版本号影响 | 定义 |
|------|------|-----------|------|
| **Type-A** | Major 变更 | MAJOR (V4.0.0) | 破坏性变更，影响现有任务/流程/智能体 |
| **Type-B** | Minor 变更 | MINOR (V3.x.0) | 向后兼容的功能增强 |
| **Type-C** | Patch 变更 | PATCH (V3.9.x) | 向后兼容的问题修复 |

---

### 1.2 Type-A 变更（Major）

**判定标准**（满足任一即为 Type-A）:
- [ ] 新增智能体
- [ ] 删除智能体
- [ ] 核心流程重构
- [ ] 宪法核心原则变更
- [ ] 智能体接口变更
- [ ] 破坏现有任务执行

---

### 1.3 Type-B 变更（Minor）

**判定标准**（满足任一即为 Type-B）:
- [ ] 新增意图类别
- [ ] 新增路由流程
- [ ] 新增技能
- [ ] 智能体职责扩展
- [ ] 配置参数调整
- [ ] 新增规范章节

---

### 1.4 Type-C 变更（Patch）

**判定标准**（默认 Type-C）:
- [ ] 文字修订
- [ ] 文档格式调整
- [ ] Bug 修复

---

## 二、流程差异

| 环节 | Type-A | Type-B | Type-C |
|------|--------|--------|--------|
| 提案模板 | 完整 RFC（10 节） | 简化提案（6 节） | Git Commit |
| 冷静期 | 3 天 | 24 小时 | 无 |
| 审批人 | 用户 + 审计 | 用户 | 自动 |

---

## 三、冷静期规则

| 变更类型 | 冷静期长度 | 起算时间 |
|----------|-----------|---------|
| **Type-A** | 3 天 | 用户确认后次日 00:00 |
| **Type-B** | 24 小时 | 用户确认后立即 |
| **Type-C** | 无 | - |

---

## 四、紧急通道

**适用场景**: 安全漏洞、严重 Bug、法律法规合规

**流程**: 简化提案 → 用户 + 审计双批准 → 立即生效 → 7 天内补充完整 RFC
