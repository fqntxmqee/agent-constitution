# 决策记录规范

**版本号**: V1.0  
**生效日期**: 2026-03-12  
**关联决策**: DEC-027 至 DEC-030

---

## 一、决策记录原则

| 原则 | 说明 |
|------|------|
| **决策必记录** | 任何决策都必须有文档记录 |
| **过程可追溯** | 讨论过程、变更理由都要记录 |
| **结果可查询** | 未来可通过关键词搜索 |
| **责任可追溯** | 明确的提案人、批准人、日期 |

---

## 二、决策记录格式

```markdown
# 决策记录 DEC-{编号}: {决策标题}

## 元数据
| 字段 | 值 |
|------|-----|
| 决策编号 | DEC-{编号} |
| 决策日期 | YYYY-MM-DD |
| 提案人 | ... |
| 批准人 | ... |

## 决策摘要
一句话描述

## 背景
问题描述

## 选项对比
| 选项 | 方案 | 优点 | 缺点 |
|------|------|------|------|
| A | ... | ... | ... |

## 讨论过程
| 时间 | 参与者 | 意见 |
|------|--------|------|
| ... | ... | ... |

## 决策结果
**采纳选项**: 选项 A
**决策理由**: ...
```

---

## 三、决策编号规则

```
DEC-{序号}
序号：3 位数字，从 001 开始递增
示例：DEC-001, DEC-002, DEC-030
```

---

## 四、决策状态流转

```
草案 → 讨论中 → 已确认 → 已实施 → (已废弃)
```

---

## 五、必须记录的内容

- [ ] 决策编号
- [ ] 决策日期
- [ ] 提案人
- [ ] 批准人
- [ ] 决策摘要
- [ ] 选项对比（至少 2 个）
- [ ] 决策理由
- [ ] 影响分析
- [ ] 讨论过程
