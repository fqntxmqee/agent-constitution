# 子 agent 进展汇报规范

**版本**: v1.0  
**创建**: 2026-03-07  
**适用范围**: 所有子 agent（DEV/TEST/DELIVERY/SPEC 等）

---

## 核心要求

**每个子 agent 必须每 3 分钟向主会话汇报一次进展**

---

## 汇报机制

### 方式 1: 使用 sessions_send（推荐）

```python
sessions_send(
    sessionKey="agent:main:main",
    message="## 🔄 进展汇报\n\n内容..."
)
```

### 方式 2: 主会话主动轮询

如果 sessions_send 不可用，主会话会每 3 分钟使用 `subagents list` 检查状态。

---

## 汇报格式

### 运行中汇报

```markdown
## 🔄 [Agent 名称] 进展同步（第 X 次）

**时间**: YYYY-MM-DD HH:MM:SS  
**状态**: 🟢 运行中  
**已运行**: X 分钟  
**消耗 Token**: X

### 当前进度

| Phase | 状态 | 说明 |
|-------|------|------|
| Phase 1 | ✅ 完成 | 项目初始化 |
| Phase 2 | 🔄 进行中 | 数据模型迁移（3/5 完成） |
| Phase 3 | ⏳ 等待 | API 接口迁移 |
| Phase 4 | ⏳ 等待 | 安全配置 |
| Phase 5 | ⏳ 等待 | E2E 测试 |

### 本周期完成的工作
- [具体完成的任务 1]
- [具体完成的任务 2]

### 下一周期计划
- [计划任务 1]
- [计划任务 2]

### 风险/阻塞
- [如有问题，说明详情]

```

### 完成汇报

```markdown
## ✅ [Agent 名称] 完成！

**状态**: 已完成  
**总运行时长**: X 分钟  
**消耗 Token**: X

### 输出文件
- [文件 1 路径]
- [文件 2 路径]

### 飞书文档
**链接**: [URL]

### 下一步
[说明下一个 Agent 的任务]
```

```markdown

**状态**: 超时/失败

[分析失败原因]

[修复建议或需要用户干预的事项]
```

## 执行流程

### 每个子 agent 的任务描述必须包含

```markdown
## 进展汇报要求

**汇报频率**: 每 3 分钟一次

**汇报方式**:
1. 使用 sessions_send(sessionKey="agent:main:main", message="...") 发送进展
2. 如 sessions_send 不可用，主会话会主动轮询

**汇报内容**:
- 当前 Phase 进度
- 本周期完成的工作
- 下一周期计划
- 风险/阻塞（如有）

**首次汇报**: 启动后 1 分钟内发送
**后续汇报**: 每 3 分钟一次
**完成汇报**: 任务完成后立即发送
```

---

## 主会话职责

### 轮询检查（备选方案）

如果 sessions_send 不可用，主会话每 3 分钟执行：

```python
subagents(action="list")
# 检查每个 agent 的状态
# 向用户发送进展汇报
```

### 汇报时间表

| 汇报 | 时间偏移 | 说明 |
|------|----------|------|
| 第 1 次 | T+1 分钟 | Agent 启动后首次汇报 |
| 第 2 次 | T+4 分钟 | 3 分钟后 |
| 第 3 次 | T+7 分钟 | 3 分钟后 |
| ... | 每 3 分钟 | 直到完成 |

## 检查清单

创建子 agent 时，必须确认：

- [ ] 任务描述中包含进展汇报要求
- [ ] 汇报频率明确（每 3 分钟）
- [ ] 汇报方式明确（sessions_send 或主会话轮询）
- [ ] 汇报格式模板已提供
- [ ] 首次汇报时间明确（启动后 1 分钟内）

---

## 违规处理

如果子 agent 未按规范汇报：

1. **主会话主动轮询** - 使用 `subagents list` 检查状态
2. **发送提醒** - 使用 `sessions_send` 或 `subagents steer` 提醒 agent
3. **记录问题** - 在汇报中说明"Agent 未按规范汇报"
