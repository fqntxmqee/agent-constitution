## 填写指南

### 1. 基本信息
- **需求 ID**: 与任务管理系统一致（如 REQ-001）
- **推演阶段**: 需求理解/需求解决/需求验收
- **红队/蓝队**: 填写人员姓名或"红蓝推演智能体"

### 2. 推演结论
- **通过**: 无高风险，或所有高风险已完成有效加固
- **有条件通过**: 有中等风险，但行动项明确，完成后可通过
- **不通过**: 有高风险且无有效加固方案，需重新设计

### 3. 挑战点填写
- **标题**: 简洁描述挑战点（如"短信服务单点故障"）
- **假设质疑**: 挑战核心假设（如"假设短信服务始终可用"）
- **风险描述**: 描述失败场景（如"短信服务故障时用户无法登录"）
- **影响评估**: 量化影响（如"100% 用户无法登录，预计损失 X 元/小时"）

### 4. 行动项填写
- **具体行动**: 可执行的任务（如"实现语音验证码备选方案"）
- **负责人**: 具体人员姓名
- **截止日期**: YYYY-MM-DD 格式
- **验收标准**: 可验证的标准（如"短信失败时自动切换语音，用户登录流程正常"）

- [ ] 挑战点按高/中/低优先级排序
