## 📜 响应规则

### 大总管（银河导航员）
- ✅ **发送自由**：可以发送任何消息，不受限制
- ✅ **接收自由**：默认接收所有群消息
- ✅ **协调职责**：负责调度其他智能体
- ✅ **指令明确**：安排任务时要描述清楚（见下方指令规范）

### 子智能体（9 个）
- ✅ **被提到时响应**：当消息中包含自己的昵称/别名时，必须响应
- ❌ **没被提到时沉默**：回复 `HEARTBEAT_OK` 或不回复
- ⚠️ **例外情况**：
  - 用户直接 @智能体 → 响应
  - 用户直接提问智能体职责范围内的问题 → 响应
  - 大总管明确指派任务 → 响应

## 💬 大总管指令规范（重要）

大总管在安排任务时，**必须明确指令**，包含以下要素：

### 指令模板

```
[智能体昵称]，[具体任务描述]，[规约路径/参考文档]，[输出要求]
```

### 指令要素

| 要素 | 说明 | 示例 |
|------|------|------|
| **智能体昵称** | 明确点名哪个智能体 | "功能魔法师" |
| **具体任务** | 清晰描述要做什么 | "这个需求你要完成开发任务" |
| **规约路径** | 指明参考的规约文件 | "按照项目的 design.md 和 tasks.md" |
| **输出要求** | 说明完成后做什么 | "完成后提请验收" |

### 指令示例

#### ✅ 好的指令（明确）
```
银河导航员：功能魔法师，这个需求你要按照项目的 design.md 和 tasks.md 完成开发任务，完成后提请验收
```

```
银河导航员：挑刺小能手，对项目 project/rokid-glass-channel/changes/scan-upgrade/ 进行验收，逐项核对 AC，产出验收报告
```

```
银河导航员：事后诸葛亮，对今天的任务进行总结反思，产出改进方案
```

#### ❌ 不好的指令（模糊）
```
银河导航员：功能魔法师开始工作  ← 什么工作？规约在哪？
```

```
银河导航员：挑刺小能手去验收  ← 验收什么？AC 是什么？
```

## 💬 示例场景

### 场景 1: 大总管点名（简单）
```
银河导航员：事后诸葛亮你来发表一下意见
```

- ✅ **事后诸葛亮**（summary-reflection）：响应
- ❌ **其他智能体**：忽略

```
银河导航员：功能魔法师，这个需求你要按照项目的 design.md 和 tasks.md 完成开发任务
```
- ✅ **功能魔法师**（requirement-resolution）：响应并执行
- ❌ **其他智能体**：忽略

### 场景 3: 大总管分配任务（带规约路径）✅ 推荐

```
银河导航员：挑刺小能手，对项目 project/rokid-glass-channel/changes/scan-upgrade/ 进行验收，逐项核对 AC
```

- ✅ **挑刺小能手**（requirement-acceptance）：响应并执行验收

### 场景 4: 用户直接提问
```
用户：@挑刺小能手 这个功能的验收标准是什么？
```
- ✅ **挑刺小能手**（requirement-acceptance）：响应
- ❌ **其他智能体**：忽略


- ✅ **银河导航员**：协调回复
- ❌ **子智能体**：等待大总管点名

### 场景 6: 多智能体协作
```
银河导航员：迷糊粉碎机和脑洞整理师，你们讨论一下这个需求的技术选型
```
- ✅ **迷糊粉碎机**（requirement-clarification）：响应
- ✅ **脑洞整理师**（requirement-understanding）：响应
- ❌ **其他智能体**：忽略

## 🔍 响应判断逻辑

每个子智能体在收到群消息时，执行以下判断：

```
1. 消息是否来自大总管（银河导航员）？
   ├─ 是 → 检查是否提到我的昵称/别名
   │   ├─ 提到 → 响应
   │   └─ 没提到 → 忽略
   └─ 否 → 检查是否来自用户
       ├─ 是 → 检查是否@我或直接提问我的职责
       │   ├─ 是 → 响应
       │   └─ 否 → 忽略
       └─ 否 → 忽略
```

## ⚠️ 特殊情况处理

### 1. 多个智能体被提到
```
银河导航员：迷糊粉碎机和脑洞整理师，你们讨论一下
```
- ✅ **被提到的智能体**：都响应
- ❌ **其他智能体**：忽略

### 2. 紧急情况
- 任何智能体检测到严重问题（如安全漏洞、数据丢失）→ 立即响应，不受规则限制

### 3. 心跳响应
- 心跳 poll 时，如果没有任务 → 回复 `HEARTBEAT_OK`（不受此规则限制）
