# 智能体协同每日自检需求 (Daily Agent Collaboration Test)

**文档类型**: 宪法规范执行保障体系 - 智能体协同验收测试  
**版本号**: V1.0  
**创建日期**: 2026-03-28  
**状态**: 待生效  
**执行时间**: 每日 24:00 (次日 00:00) 自动执行

## 一、测试目标

通过一个**真实可执行的需求场景**，验证所有智能体能否按宪法规范 V3.16.0 协同工作，确保：

1. **流程完整性**: 澄清 → 理解 → 解决 → 验收 → 交付 完整执行
2. **智能体协同**: 9 大智能体各司其职，Hub-Spoke 模式正常
3. **规约先行**: 每个阶段产出符合规范
4. **Hard Gate 检查**: 4 个 Gate 全部触发并记录
5. **审计监督**: 实时熔断 + 定时审计正常
6. **自动交付**: 无需人工干预，自动完成全流程

---

## 二、测试需求设计

### 2.1 需求场景：创建"每日状态报告"文件

**需求描述**: 
创建一个简单的 Markdown 文件，记录当日系统状态，包含：
- 日期时间
- 运行智能体数量
- 当日完成任务数
- 系统健康状态

**选择理由**:
- ✅ 足够简单：5 分钟内可完成，不影响正常任务
- ✅ 足够完整：需要所有智能体参与
- ✅ 可自动化：文件创建 + Git 提交可自动执行
- ✅ 有验收标准：文件存在 + 内容完整 + Git 提交成功
- ✅ 可追溯：每日产出可累积成历史报告

### 2.2 需求复杂度评级：**C 级（简单）**

| 维度 | 评级 | 说明 |
|------|------|------|
| 功能点数量 | 1 个 | 创建单个文件 |
| 技术复杂度 | 低 | Markdown 文件 + Git 提交 |
| 依赖外部系统 | 无 | 纯本地操作 |
| 预计耗时 | 5-10 分钟 | 快速流程 |

## 三、智能体职责与产出

| 阶段 | 智能体 | 昵称 | 职责 | 产出物路径 |
|------|--------|------|------|--------|
| **澄清** | requirement-clarification | 需求澄清 🎯 | 理解需求意图，生成提案 | `.tasks/requirement-clarification/REQ-{ID}/proposal.md` |
| **理解** | requirement-understanding | 脑洞整理师 💡 | 设计执行蓝图 | `.tasks/requirement-understanding/REQ-{ID}/specs/*.md` |
| **解决** | requirement-resolution | 功能魔法师 🪄 | 执行文件创建 | `.tasks/requirement-resolution/REQ-{ID}/daily-status-YYYYMMDD.md` + Git 提交 |
| **验收** | requirement-acceptance | 挑刺小能手 🔍 | 验证产出物 | `.tasks/requirement-acceptance/REQ-{ID}/acceptance-report.md` |
| **交付** | requirement-delivery | 最后一公里 📦 | 最终交付确认 | `.tasks/requirement-delivery/REQ-{ID}/delivery-report.md` |
| **辅助** | debugger | 调试专家 🔬 | 可调试性审查（触发点：解决阶段失败时） | `debug-report.md` (如有需要) |
| **辅助** | red-team-simulation | 红蓝推演师 🎭 | 方案挑战（C 级任务可选） | `red-team-report.md` (可选) |
| **监督** | audit | 规则守护者 🛡️ | 全流程合规监控 | `fuse-poll-YYYY-MM-DD.jsonl` |
| **总结** | summary-reflection | 事后诸葛亮 📝 | 执行后复盘分析 | `daily-collab-test-summary.md` |
| **协调** | main | 银河导航员 🧭 | 任务分发 + 进度跟踪 | `.tasks/index.md` 更新 |

## 五、验收标准 (Checklist)

### 5.1 流程完整性

- [ ] 需求澄清阶段完成（proposal.md 存在）
- [ ] 需求理解阶段完成（specs/ 目录完整）
- [ ] 需求解决阶段完成（文件创建 + Git 提交）
- [ ] 需求验收阶段完成（acceptance-report.md 存在）
- [ ] 需求交付阶段完成（delivery-report.md 存在）

### 5.2 Hard Gate 检查

- [ ] Gate #1 记录存在（澄清→理解）
- [ ] Gate #2 记录存在（理解→解决）
- [ ] Gate #3 记录存在（解决→验收）
- [ ] Gate #4 记录存在（验收→交付）

### 5.3 产出物验证

- [ ] `daily-status-YYYYMMDD.md` 文件存在
- [ ] 文件内容包含：日期时间、智能体数量、任务数、健康状态
- [ ] Git 提交记录存在（commit message 包含 REQ-DAILY-TEST）
- [ ] `.tasks/index.md` 状态更新为 completed

### 5.4 审计监督

- [ ] 实时熔断监控日志存在（fuse-poll-*.jsonl）
- [ ] 定时审计报告存在（audit-*.md）
- [ ] 无违规记录（VIO-001~007 全部通过）

### 5.5 总结反思

- [ ] `daily-collab-test-summary.md` 存在
- [ ] 包含量化指标（耗时、成功率等）
- [ ] 包含改进建议（如有问题）

## 六、定时任务配置

### 6.1 Cron 配置

**配置位置**: `crontab -e`

**配置内容**:
```bash
# 智能体协同每日自检 - 每日 24:00 (00:00) 执行
0 0 * * * cd /Users/fukai/project/openclaw-workspace && openclaw agent --agent main --message "执行每日智能体协同自检 REQ-DAILY-TEST-$(date +\%Y\%m\%d)" >> agents/constitution/audit/logs/daily-collab-test.log 2>&1
```

### 6.2 执行脚本

**脚本位置**: `scripts/daily-collab-test.sh`

```bash
#!/bin/bash
# daily-collab-test.sh - 智能体协同每日自检执行脚本

set -e

DATE=$(date +%Y%m%d)
TASK_ID="REQ-DAILY-TEST-${DATE}"

echo "=========================================="
echo "智能体协同每日自检"
echo "任务 ID: ${TASK_ID}"
echo "执行时间: $(date '+%Y-%m-%d %H:%M:%S')"
echo "=========================================="

# 通过 OpenClaw 主智能体执行
openclaw agent --agent main --message "执行每日智能体协同自检 ${TASK_ID}"

echo "自检执行完成"
```

### 6.3 手动触发

```bash
# 手动执行
bash scripts/daily-collab-test.sh

# 或直接用 OpenClaw
openclaw agent --agent main --message "执行每日智能体协同自检 REQ-DAILY-TEST-$(date +%Y%m%d)"
```

## 七、失败处理

### 7.1 失败场景与处理

| 失败场景 | 可能原因 | 处理方案 |
|----------|----------|----------|
| 需求澄清超时 (>10min) | 智能体会话卡住 | 银河导航员检测超时，发送告警 |
| 文件创建失败 | 权限问题/磁盘满 | 功能魔法师重试 3 次，失败则告警 |
| Git 提交失败 | 网络问题/冲突 | 重试 3 次，仍失败则标记 blocked |
| 验收不通过 | 产出物不完整 | 挑刺小能手返回重做 |
| Hard Gate 未触发 | 流转检查逻辑错误 | 规则守护者熔断 + 告警 |

### 7.2 告警机制

**飞书告警格式**:
```markdown
🚨 每日智能体协同自检失败

**任务 ID**: REQ-DAILY-TEST-20260328
**失败阶段**: {阶段名称}
**失败智能体**: {智能体昵称}
**错误信息**: {错误详情}

**处理建议**:
1. 查看日志：agents/constitution/audit/logs/daily-collab-test.log
2. 检查失败原因
3. 手动重新执行或修复后等待下次自动执行

**快速操作**:
- [查看日志](link)
- [手动重试](link)
- [跳过今日](link)
```
