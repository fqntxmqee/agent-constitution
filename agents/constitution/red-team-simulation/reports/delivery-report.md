## 📋 交付概述

### 智能体定位

红蓝推演智能体是 OpenClaw 银河导航员团队的**第 9 大智能体**，担任"魔鬼代言人"和"多视角分析器"角色。

**核心价值**:
- 🔴 **红队挑战**: 质疑假设、识别风险、提出替代方案
- 🔵 **蓝队完善**: 应对建议、方案加固、验收完善
- 📊 **决策辅助**: 为银河导航员提供多维度分析

---

## 🎯 核心功能

### 红队挑战（7 个维度）

| 维度 | 说明 | 示例问题 |
|------|------|---------|
| 假设挑战 | 质疑基础假设 | "这个假设一定成立吗？" |
| 风险识别 | 识别技术/业务风险 | "单点故障在哪里？" |
| 替代方案 | 提出更优方案 | "有没有更简单的实现方式？" |
| 边界条件 | 分析极端情况 | "用户量增长 10 倍会怎样？" |
| 性能扩展 | 评估性能和可扩展性 | "响应时间能满足 SLA 吗？" |
| 依赖分析 | 分析外部依赖风险 | "依赖服务不可用时怎么办？" |
| 用户体验 | 从用户角度挑战 | "用户能理解这个交互吗？" |

### 蓝队完善（5 个策略）

| 策略 | 说明 | 输出示例 |
|------|------|---------|
| 风险缓解 | 降低风险发生概率 | "增加超时 + 重试 + 降级" |
| 方案设计 | 增加冗余/降级/回滚 | "主从架构 + 自动切换" |
| 监控降级 | 增加可观测性 | "Prometheus 监控 + 告警" |
| 备选方案 | 提供 Plan B | "如果主方案不可行，可以..." |
| 验收完善 | 补充测试场景 | "增加边界测试/压力测试" |

## 📚 使用指南

### 快速开始

```bash
# 方式 1: CLI 直接调用
openclaw agent --agent red-team-simulation --channel webchat --message "请对以下设计进行红蓝推演..."

# 方式 2: 银河导航员自动调用
# 当任务复杂度≥B 级时，在需求理解/方案评审阶段自动触发
```

### 输入示例

```markdown
请对以下设计方案进行红蓝推演分析：

## 需求概述

- 架构：...
- 数据库：...
- 缓存策略：...

## 已知限制
- ...
```
