# FitBot 健身教练智能体重构 - 需求规格说明书

## 1. 功能需求 (Functional Requirements)

### 1.1 M1: 运动规划模块 (P0)

#### FR-001: 三分化力量训练计划

**描述:** 生成个性化的三分化力量训练计划

**输入:**
- 用户健身水平 (中级)
- 可用设备 (哑铃)
- 可用时间 (每次 45-60 分钟)
- 训练频率 (3 次/周)

**输出:**
- 每周训练计划
- 每日训练动作列表 (推/拉/腿)
- 组数/次数/重量建议
- 休息间隔建议

**验收条件:**
- [ ] 计划符合三分化原则 (推/拉/腿)
- [ ] 动作适合家庭训练
- [ ] 强度适合中级水平
- [ ] 可通过飞书消息查看

---

#### FR-002: 跑步训练计划

**描述:** 生成半马提升训练计划

**输入:**
- 当前成绩 (半马 2:10)
- 目标成绩 (半马 2:00)
- 可用训练时间

**输出:**
- 每周跑步计划
- 配速建议
- 训练类型 (间歇/耐力/恢复)
- 赛前减量计划

**验收条件:**
- [ ] 计划科学有效
- [ ] 符合半马训练原则
- [ ] 考虑恢复时间
- [ ] 可通过飞书消息查看

---

#### FR-003: 骑车训练安排

**描述:** 安排日常骑车训练

**输入:**
- 日常骑车习惯 (1 小时/天)
- 心率数据 (如有)
- 路线/地形

**输出:**
- 骑车强度建议
- 心率区间指导
- 与力量训练协调

**验收条件:**
- [ ] 与力量训练协调
- [ ] 不影响力量恢复
- [ ] 心率区间合理

---

#### FR-004: 训练日程协调

**描述:** 协调力量/跑步/骑车的训练日程

**输入:**
- 三项训练计划
- 用户可用时间
- 恢复状态

**输出:**
- 综合训练日历
- 冲突检测与解决
- 恢复日安排

**验收条件:**
- [ ] 无训练冲突
- [ ] 恢复日充足
- [ ] 负荷分布合理
- [ ] 可通过飞书消息查看

---

### 1.2 M2: 饮食规划模块 (P0)

#### FR-005: 高蛋白饮食计划

**描述:** 生成高蛋白饮食计划

**输入:**
- 用户体重/体脂
- 训练目标 (增肌)
- 饮食偏好

**输出:**
- 每日热量目标
- 蛋白质摄入目标 (g/kg)
- 三餐建议
- 加餐建议

**验收条件:**
- [ ] 蛋白质摄入充足 (1.6-2.2g/kg)
- [ ] 热量平衡合理
- [ ] 食物选择可行
- [ ] 可通过飞书消息查看

---

#### FR-006: 训练日/休息日差异

**描述:** 区分训练日和休息日饮食

**输入:**
- 当日训练类型
- 训练强度
- 训练时长

**输出:**
- 训练日饮食计划
- 休息日饮食计划
- 碳水循环建议

**验收条件:**
- [ ] 训练日碳水充足
- [ ] 休息日热量适度降低
- [ ] 蛋白质摄入稳定

---

#### FR-007: 跑步前后营养

**描述:** 跑步训练前后营养建议

**输入:**
- 跑步时间
- 跑步强度
- 跑步时长

**输出:**
- 跑前饮食建议
- 跑后恢复饮食
- 补液建议

**验收条件:**
- [ ] 跑前能量充足
- [ ] 跑后恢复及时
- [ ] 电解质补充合理

- 改善建议
- 与训练关联分析

**验收条件:**
- [ ] 评分科学合理
- [ ] 建议可执行
- [ ] 关联性分析准确

---

#### FR-012: 恢复状态评估

**描述:** 综合评估恢复状态

**输出:**
- 恢复状态评分

- 风险提示

**验收条件:**
- [ ] 评估综合全面

- [ ] 风险预警准确

## 2. 非功能需求 (Non-Functional Requirements)

### 2.1 性能需求

| 指标 | 要求 | 测量方式 |
|------|------|----------|
| 消息响应时间 | <2 秒 | 端到端时间 |
| 数据导入时间 | <10 秒 | 1000 条记录 |
| 周报生成时间 | <30 秒 | 完整周报 |
| 系统可用性 | 99% | 月度统计 |

---

### 2.2 安全需求

- 用户数据加密存储
- 飞书 token 安全管理
- 数据访问控制
- 隐私保护

---

### 2.3 可用性需求

- 飞书消息界面友好
- 数据导入简单
- 报告易读
- 操作便捷

---

### 2.4 兼容性需求

| 组件 | 要求 |
|------|------|
| OpenClaw | 最新稳定版 |
| 飞书 | 机器人 API |
| 华为健康 | CSV/JSON 导出 |
| Node.js | ≥18.0.0 |

## 3. 数据需求

### 3.1 用户数据

| 字段 | 类型 | 说明 |
|------|------|------|
| userId | string | 用户 ID (飞书) |
| fitnessLevel | string | 健身水平 |
| weight | number | 体重 (kg) |
| bodyFat | number | 体脂率 (%) |
| height | number | 身高 (cm) |
| age | number | 年龄 |
| gender | string | 性别 |

---

### 3.2 训练数据

| 字段 | 类型 | 说明 |
|------|------|------|
| date | string | 日期 |
| type | string | 训练类型 |
| duration | number | 时长 (分钟) |
| calories | number | 卡路里 |
| avgHeartRate | number | 平均心率 |
| maxHeartRate | number | 最大心率 |
| details | object | 训练详情 |

---

### 3.3 饮食数据

| 字段 | 类型 | 说明 |
