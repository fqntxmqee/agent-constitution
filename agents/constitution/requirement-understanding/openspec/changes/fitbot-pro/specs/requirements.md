# FitBot Pro 健身教练智能体 - 需求规格说明书

## 1. 功能需求 (Functional Requirements)

### 1.1 运动规划模块

#### FR-001: 三分化力量训练计划

**描述:** 生成个性化的三分化力量训练计划

**输入:**
- 用户健身水平 (中级)
- 可用设备 (哑铃)
- 可用时间 (每次 45-60 分钟)
- 训练频率 (3 次/周)

**输出:**
- 每周训练计划
- 每日训练动作列表
- 组数/次数/重量建议
- 休息间隔建议

**验收条件:**
- 计划符合三分化原则 (推/拉/腿)
- 动作适合家庭训练
- 强度适合中级水平

---

#### FR-002: 跑步训练计划

**描述:** 生成半马提升训练计划

**输入:**
- 当前成绩 (半马 2:10)
- 目标成绩 (半马 2:00)
- 可用训练时间

**输出:**
- 每周跑步计划
- 配速建议
- 训练类型 (间歇/耐力/恢复)
- 赛前减量计划

**验收条件:**
- 计划科学有效
- 符合半马训练原则
- 考虑恢复时间

---

#### FR-003: 骑车训练安排

**描述:** 安排日常骑车训练

**输入:**
- 日常骑车习惯 (1 小时/天)
- 心率数据
- 路线/地形

**输出:**
- 骑车强度建议
- 心率区间指导
- 恢复建议

**验收条件:**
- 与力量训练协调
- 不影响力量恢复
- 心率区间合理

---

#### FR-004: 训练日程协调

**描述:** 协调力量/跑步/骑车的训练日程

**输入:**
- 三项训练计划
- 用户可用时间
- 恢复状态

**输出:**
- 综合训练日历
- 冲突检测与解决
- 恢复日安排

**验收条件:**
- 无训练冲突
- 恢复日充足
- 负荷分布合理

---

### 1.2 饮食规划模块

#### FR-005: 高蛋白饮食计划

**描述:** 生成高蛋白饮食计划

**输入:**
- 用户体重/体脂
- 训练目标 (增肌)
- 饮食偏好

**输出:**
- 每日热量目标
- 蛋白质摄入目标 (g/kg)
- 三餐建议
- 加餐建议

**验收条件:**
- 蛋白质摄入充足 (1.6-2.2g/kg)
- 热量平衡合理
- 食物选择可行

---

#### FR-006: 训练日/休息日差异

**描述:** 区分训练日和休息日饮食

**输入:**
- 当日训练类型
- 训练强度
- 训练时长

**输出:**
- 训练日饮食计划
- 休息日饮食计划
- 碳水循环建议

**验收条件:**
- 训练日碳水充足
- 休息日热量适度降低
- 蛋白质摄入稳定

---

#### FR-007: 跑步前后营养

**描述:** 跑步训练前后营养建议

**输入:**
- 跑步时间
- 跑步强度
- 跑步时长

**输出:**
- 跑前饮食建议
- 跑后恢复饮食
- 补液建议

**验收条件:**
- 跑前能量充足
- 跑后恢复及时
- 电解质补充合理

---

### 1.3 数据分析模块

#### FR-008: 华为手表数据导入

**描述:** 导入华为手表运动数据

**输入:**
- 华为健康导出数据
- 或 API 同步数据

**输出:**
- 数据导入确认
- 数据解析结果
- 异常数据标记

**验收条件:**
- 支持 CSV/JSON 导入
- 数据解析准确
- 异常数据处理

---

#### FR-009: 训练负荷分析

**描述:** 分析训练负荷数据

**输入:**
- 训练记录
- 心率数据
- 主观疲劳度

**输出:**
- 训练负荷趋势
- 急性/慢性负荷比 (ACWR)
- 过度训练风险

**验收条件:**
- 负荷计算准确
- 趋势可视化
- 风险预警及时

---

#### FR-010: 心率数据分析

**描述:** 分析心率数据

**输入:**
- 静息心率
- 运动心率
- 心率变异性 (HRV)

**输出:**
- 心率区间分析
- 恢复状态评估
- 训练建议

**验收条件:**
- 区间划分准确
- 恢复评估可靠
- 建议合理

---

#### FR-011: 睡眠数据分析

**描述:** 分析睡眠质量数据

**输入:**
- 睡眠时长
- 睡眠阶段 (深睡/浅睡/REM)
- 睡眠规律

**输出:**
- 睡眠质量评分
- 改善建议
- 与训练关联分析

**验收条件:**
- 评分科学合理
- 建议可执行
- 关联性分析准确

---

#### FR-012: 恢复状态评估

**描述:** 综合评估恢复状态

**输入:**
- 训练负荷
- 睡眠质量
- 心率变异性
- 主观疲劳

**输出:**
- 恢复状态评分
- 训练建议 (训练/恢复/休息)
- 风险提示

**验收条件:**
- 评估综合全面
- 建议科学合理
- 风险预警准确

---

## 2. 非功能需求 (Non-Functional Requirements)

### 2.1 性能需求

| 指标 | 要求 |
|------|------|
| 消息响应时间 | <2 秒 |
| 数据同步时间 | <10 秒 |
| 周报生成时间 | <30 秒 |
| 系统可用性 | 99% |

### 2.2 安全需求

- 用户数据加密存储
- API 密钥安全管理
- 数据访问控制
- 隐私保护

### 2.3 可用性需求

- 飞书消息界面友好
- 数据导入简单
- 报告易读
- 操作便捷

### 2.4 兼容性需求

| 组件 | 要求 |
|------|------|
| 华为手表 | 支持数据导出/API |
| 飞书 | 机器人 API |
| 数据存储 | SQLite/JSON |

## 3. 数据需求

### 3.1 用户数据

| 字段 | 类型 | 说明 |
|------|------|------|
| userId | string | 用户 ID |
| fitnessLevel | string | 健身水平 |
| weight | number | 体重 (kg) |
| bodyFat | number | 体脂率 (%) |
| height | number | 身高 (cm) |
| age | number | 年龄 |
| gender | string | 性别 |

### 3.2 训练数据

| 字段 | 类型 | 说明 |
|------|------|------|
| date | string | 日期 |
| type | string | 训练类型 |
| duration | number | 时长 (分钟) |
| calories | number | 卡路里 |
| avgHeartRate | number | 平均心率 |
| maxHeartRate | number | 最大心率 |
| details | object | 训练详情 |

### 3.3 饮食数据

| 字段 | 类型 | 说明 |
|------|------|------|
| date | string | 日期 |
| meals | array | 餐食记录 |
| protein | number | 蛋白质 (g) |
| carbs | number | 碳水 (g) |
| fat | number | 脂肪 (g) |
| calories | number | 总热量 |
