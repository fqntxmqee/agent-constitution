## 3. 截图规范

### 3.1 命名规范

```
格式：{类别}-{场景}-{序号}.png
示例：E2E-SCR-001.png, INT-SCR-002.png, PERF-SCR-003.png
```

### 3.2 截图要求

- **分辨率:** 1920x1080 或更高
- **格式:** PNG (无损)
- **内容:** 清晰显示关键信息
- **标注:** 必要时添加箭头/文字标注

### 3.3 存储路径

```
phase2/implementation/reports/screenshots/
├── e2e/
│   ├── E2E-SCR-001.png
│   ├── E2E-SCR-002.png
│   └── ...
├── integration/

│   └── ...
└── performance/
    ├── PERF-SCR-001.png
    ├── PERF-SCR-002.png
    └── ...
```

## 4. 截图执行步骤

### 4.1 准备工作

```bash
# 1. 创建截图目录
mkdir -p phase2/implementation/reports/screenshots/{e2e,integration,performance}

# 2. 准备测试环境
npm install
npm run build

# 3. 启动监控服务
npm run monitor
```

### 4.2 执行截图

```bash
# E2E 测试截图
npm run test:e2e -- --screenshot

# 集成测试截图
npm run test:integration -- --screenshot

# 性能监控截图
npm run test:performance -- --screenshot
```

### 4.3 截图整理

```bash
# 1. 检查截图完整性
ls -la phase2/implementation/reports/screenshots/

# 2. 重命名截图 (如需要)
mv screenshot-001.png E2E-SCR-001.png

# 3. 生成截图索引
npm run generate:screenshot-index
```
