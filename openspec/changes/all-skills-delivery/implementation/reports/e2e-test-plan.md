## 测试目标

验证智能体技能在完整工作流程中的协同能力，确保从需求输入到最终交付的端到端流程畅通。

---

## 测试场景

### 场景 1: 标准构建流验证

**流程**: 需求澄清 → 需求理解 → 需求解决 → 需求验收 → 交付

**测试步骤**:
1. 输入模糊需求
2. Skill-03 检测模糊性并生成澄清问题
3. Skill-05 生成执行蓝图
4. Skill-06/Skill-07 执行蓝图
5. Skill-08 进行验收测试
6. 交付最终产物

**预期结果**:
- 每个环节数据正确传递
- 无数据丢失或格式错误
- 最终交付物符合验收标准

---

### 场景 2: 快速执行流验证

**流程**: 需求澄清 → 需求解决 → 需求验收 → 交付

**测试步骤**:
1. 输入清晰需求
2. Skill-03 确认无模糊性
3. Skill-07 直接执行轻量计划
4. Skill-08 进行验收测试
5. 交付最终产物

**预期结果**:
- 跳过理解环节，直接进入执行
- 执行效率提升
- 交付质量不变

---

### 场景 3: 复合意图处理验证

**流程**: 多意图识别 → 任务分解 → 并行执行 → 结果聚合

**测试步骤**:
1. 输入包含多个意图的复合需求
2. Skill-01 识别多个意图
3. Skill-04 路由到不同执行智能体
4. 并行执行多个任务
5. 聚合执行结果

**预期结果**:
- 所有意图被正确识别
- 任务正确分解和路由
- 结果正确聚合
