## 1. 项目概述

### 1.1 背景

V3.7 宪法规范定义了 38 个智能体技能，支撑完整的标准构建流（澄清→理解→解决→验收→交付）及支撑智能体（进展跟进、审计、总结反思）。

### 1.2 范围

- **阶段一**: 12 个 P0 技能验收交付
- **阶段二**: 26 个 P1+P2 技能开发，完成全部 38 个技能

### 1.3 术语表

| 术语 | 定义 |
|------|------|
| P0 | 核心流程必需技能（10 个） |
| P1 | 增强功能技能（14 个） |
| P2 | 监控与审计技能（14 个） |
| OpenSpec | 标准化执行蓝图（proposal/specs/design/tasks） |
| AC | 验收标准（Acceptance Criteria） |

## 3. 功能需求（Functional Requirements）

### 3.1 P0 核心技能需求

#### FR-P0-001: Skill-01 全域意图分类引擎

**描述**: 将用户请求分类为开发类/内容类/技能类/运维类，识别主意图与从属意图

**输入**: 用户原始表述

**输出**:
```json
{
  "primaryIntent": "development",
  "secondaryIntents": ["integration"],
  "confidence": 0.95,
  "reasoning": "..."
}
```

**验收标准**:
- AC-01-01: 意图分类准确率 ≥90%
- AC-01-02: 支持复合意图识别
- AC-01-03: 响应时间 <2 秒
- AC-01-04: 输出 JSON 格式有效

#### FR-P0-002: Skill-03 跨域模糊性探测器

**描述**: 检测需求中的模糊点，生成追问清单

**输入**: 用户请求 + 意图分类结果

**输出**:
```json
{
  "ambiguities": [...],
  "isClear": true/false,
  "clarificationQuestions": [...]
}
```

**验收标准**:
- AC-03-01: 模糊点检出率 ≥80%
- AC-03-02: 追问问题清晰具体
- AC-03-03: 支持多轮追问
- AC-03-04: 响应时间 <3 秒

#### FR-P0-003: Skill-04 动态路由决策器

**描述**: 决定走标准构建流还是快速执行流

**输入**: 意图分类 + 模糊性检测结果

**输出**:
```json
{
  "route": "standard|fast",
  "complexity": "low|medium|high",
  "reasoning": "..."
}
```

**验收标准**:
- AC-04-01: 路由规则可配置
- AC-04-02: 决策理由清晰
- AC-04-03: 支持用户覆盖

#### FR-P0-004: Skill-06 多模态蓝图转换器

**描述**: 将已确认需求转化为标准化执行蓝图

**输入**: 《已确认提案》

**输出**:
- 开发类：proposal.md, specs/requirements.md, design.md, tasks.md
- 内容类：内容执行大纲
- 技能类：参数映射与执行计划

**验收标准**:
- AC-06-01: 模板完整、结构清晰
- AC-06-02: 支持多种蓝图形态
- AC-06-03: 生成的任务可执行、可验证

#### FR-P0-005: Skill-09 跨域验收标准定义器

**描述**: 为任务定义可验证的 AC（验收标准）

**输入**: 执行蓝图

**输出**: 验收标准列表（可验证、可量化）

**验收标准**:
- AC-09-01: AC 可验证、可量化
- AC-09-02: 覆盖所有关键功能
- AC-09-03: 验证方法明确

#### FR-P0-006: Skill-10 任务原子化拆解器

**描述**: 将蓝图/计划拆解为可独立验证的步骤

**输入**: 《执行蓝图》或《轻量执行计划》

**输出**: Step-by-Step 执行步骤

**验收标准**:
- AC-10-01: 步骤独立、可验证
- AC-10-02: 步骤间依赖清晰
- AC-10-03: 每步有明确输入输出

#### FR-P0-007: Skill-13 自适应测试/审查生成器

**描述**: 根据代码/需求生成测试用例

**输入**: 代码 + 需求规范

**输出**: 测试文件（Jest/Pytest 等）

**验收标准**:
- AC-13-01: 测试覆盖关键功能
- AC-13-02: 测试可运行
- AC-13-03: 测试用例清晰

#### FR-P0-008: Skill-16 自动化测试/审查运行器

**描述**: 运行测试套件、代码审查、安全扫描

**输入**: 代码 + 测试文件

**输出**: 测试报告

**验收标准**:
- AC-16-01: 测试报告完整
- AC-16-02: 支持覆盖率统计
- AC-16-03: 支持截图取证

#### FR-P0-009: Skill-18 事实与逻辑验证器

**描述**: 联网检索、事实核查、交叉验证

**输入**: 待验证内容

**输出**:
```json
{
  "verified": true/false,
  "sources": [...],
  "contradictions": [...]
}
```

**验收标准**:
- AC-18-01: 支持多源交叉验证
- AC-18-02: 置信度评估准确
- AC-18-03: 发现矛盾点

#### FR-P0-010: Skill-25 敏感信息终检器

**描述**: 交付前敏感信息扫描（API Key、密码、PII）

**输入**: 代码目录

**输出**:
```json
{
  "passed": true/false,
  "findings": [...]
}
```

**验收标准**:
- AC-25-01: 检测准确率高
- AC-25-02: 误报率低
- AC-25-03: 报告清晰

- NFR-007: 每个技能有完整文档（SKILL.md + README.md）
